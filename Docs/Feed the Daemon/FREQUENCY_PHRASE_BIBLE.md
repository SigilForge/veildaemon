# FEED THE DAEMON — FREQUENCY PHRASE BIBLE

**Status:** Required content artifact · structure stub
**Required before:** Proof 2 — First Mutation Loop
**Authority:** `SHEEP_NODE_MVP_SPEC.md` defines the implementation contract; this file defines the authored musical grammar that contract consumes.

No Frequency transformation is approved merely because it has atmospheric copy or a different filter preset. Each sound-role × Frequency pairing must be audibly distinct, bounded, reproducible, and attributable to its source.

## Required Frequencies

- Dream
- Hunger
- Silence
- Stillness
- Empyrean
- Becoming

These names must remain aligned with the live VeilDaemon Operator Frequency taxonomy.

## Required base roles

- Kick
- Snare
- Bass
- Voice
- Pad
- Noise / Rain
- Pulse

## Entry contract

Every Frequency × role entry must define:

- phrase function
- timing behavior
- buildup behavior
- omission behavior
- release behavior
- interaction with duplicate performers
- decay behavior
- visual behavior
- mutation boundaries
- authored fallback behavior
- audible identity test
- source asset and asset-pack version

## Acceptance rule

Two transformations are not complete merely because their descriptions differ. They must sound recognizably different without requiring a label while remaining compatible with the bounded ensemble and deterministic replay model.

## Existing seed examples

The MVP specification currently seeds two examples:

- Noise / Rain + Stillness: **build → suspend → breach**
- Noise / Rain + Silence: **signal → omission → displaced return**

Expand those examples here before implementation; do not fork their meaning into a second undocumented grammar.
